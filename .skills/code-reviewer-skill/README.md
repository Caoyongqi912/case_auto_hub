# Code Reviewer Skill - 代码审查技能

## 📖 简介

Code Reviewer Skill 是一个全面的代码审查技能，能够对代码进行8个维度的质量分析，自动生成包含流程图的标准化审查文档，并提供可直接执行的修改建议。

## 🎯 什么情况下使用

### 适用场景

#### 1. 代码质量审查
- **代码提交前审查**: 在提交代码到版本控制系统之前，进行全面的代码质量检查
- **代码合并前审查**: 在合并分支或 Pull Request 时，确保代码质量符合标准
- **定期代码审计**: 定期对项目代码进行质量审计，发现潜在问题

#### 2. 问题诊断与修复
- **Bug 排查**: 当代码出现问题时，使用该技能进行全面审查，找出根本原因
- **性能优化**: 识别代码中的性能瓶颈和资源浪费
- **安全漏洞检测**: 发现代码中的安全漏洞和风险点

#### 3. 代码重构与优化
- **重构前评估**: 在重构代码前，全面了解代码结构和问题
- **架构优化**: 识别架构设计问题，提供优化建议
- **代码规范化**: 统一代码风格，提高代码可读性

#### 4. 学习与培训
- **代码学习**: 通过流程图快速理解复杂代码的执行逻辑
- **最佳实践学习**: 从审查建议中学习代码最佳实践
- **团队培训**: 作为团队代码审查的标准化工具

#### 5. 项目交接
- **代码交接**: 在项目交接时，生成详细的代码分析文档
- **新人入职**: 帮助新成员快速了解项目代码结构
- **文档生成**: 自动生成代码质量报告，减少手动编写文档的工作量

### 典型使用案例

#### 案例 1: 发现 SQL 注入漏洞
```
场景: 审查数据库操作代码
问题: 代码中存在 SQL 注入风险
结果: 技能识别出问题并提供参数化查询的修复方案
```

#### 案例 2: 优化循环性能
```
场景: 审查包含循环的代码
问题: 循环中重复查询数据库，性能低下
结果: 技能识别 N+1 查询问题，提供批量查询或缓存的优化建议
```

#### 案例 3: 理解复杂业务逻辑
```
场景: 接手他人的复杂业务代码
问题: 代码逻辑复杂，难以理解
结果: 技能生成流程图，清晰展示执行路径和分支逻辑
```

## 🚀 如何使用

### 基本使用方法

#### 1. 审查单个文件

**输入**:
```
审查文件 common/mysqlClient.py 的代码质量
```

**输出**:
- 包含中文注释的代码执行流程图
- 8个维度的代码质量分析
- 可直接执行的修改建议
- 完整的审查报告（Markdown 格式）

#### 2. 审查目录

**输入**:
```
请审查 common 目录下的所有代码文件
```

**输出**:
- 目录下所有代码文件的审查报告
- 每个文件的流程图和质量分析
- 汇总的问题统计和优先级建议

#### 3. 审查代码片段

**输入**:
```
审查以下 Python 代码：

def get_user_data(user_id):
    query = "SELECT * FROM users WHERE id = " + user_id
    result = db.execute(query)
    return result
```

**输出**:
- 代码片段的流程图
- 安全问题识别（SQL 注入）
- 功能问题分析
- 修复建议和优化方案

### 高级使用方法

#### 1. 生成报告并保存

**步骤**:
1. 使用技能审查代码
2. 将输出保存为 Markdown 文件
3. 在 IDE 或浏览器中查看报告

**示例**:
```
审查文件 croe/interface/runner.py，并将结果保存到 code_review_report.md
```

#### 2. AI 自动修复

**步骤**:
1. 生成包含修复建议的审查报告
2. AI 助手读取报告
3. 根据修复建议自动应用代码修改
4. 运行测试验证修复效果

**示例**:
```
1. 生成审查报告
2. AI 助手读取报告中的 "Actionable Recommendations" 部分
3. 按照优先级应用修复建议
4. 运行测试确保没有回归问题
```

#### 3. 团队协作审查

**步骤**:
1. 团队成员各自审查负责的模块
2. 汇总审查报告
3. 团队讨论关键问题
4. 制定修复计划

### 使用技巧

#### 技巧 1: 关注高优先级问题
审查报告按优先级分类问题：
- **Critical**: 必须立即修复（安全漏洞、功能缺陷）
- **Important**: 应该尽快修复（性能问题、可读性问题）
- **Minor**: 建议修复（代码风格、规范问题）

#### 技巧 2: 利用流程图理解代码
流程图提供：
- 清晰的执行路径可视化
- 中英文双语注释
- 决策点和分支逻辑标注
- 异常处理路径展示

#### 技巧 3: 使用检查清单跟踪进度
报告中的 "Implementation Checklist" 帮助：
- 跟踪修复进度
- 确保不遗漏重要修复
- 验证修复效果

## 📋 审查维度说明

该技能从8个维度对代码进行全面审查：

### 1. 功能正确性（核心维度）
- 代码执行验证
- 逻辑正确性检查
- 边界条件处理
- 异常场景捕获
- 资源问题检测
- 测试覆盖评估

### 2. 安全性评估（线上风险优先）
- 注入漏洞识别
- 访问控制验证
- 敏感数据保护
- 日志安全检查
- 输入验证评估

### 3. 性能与资源占用
- 数据库操作优化
- API 调用优化
- 计算复杂度分析
- 资源管理检查
- 缓存策略评估

### 4. 代码可读性与可维护性
- 命名质量审查
- 函数设计评估
- 注释质量检查
- 代码结构分析
- DRY 原则验证

### 5. 架构与设计规范
- 架构模式验证
- 耦合度分析
- 可扩展性评估
- 代码复用检查

### 6. 编码规范与风格
- 格式一致性检查
- 命名规范验证
- Lint 合规性检查
- 死代码识别

### 7. 兼容性与鲁棒性
- 多环境支持评估
- 错误容错能力
- 依赖管理检查
- 降级方案验证

### 8. 业务逻辑准确性
- 业务规则验证
- 状态流转检查
- 需求理解准确性
